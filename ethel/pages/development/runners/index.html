Runner development

When starting a new runner, look for the most recently updated runner of the type you're working on to use as a reference.

All runners have access to a number of useful functions via the task and logging services as well as the BaseRunner class.

Task runner base class

  • config - the full job config object
  • hashGet - creates a hash of the value supplied
  • logger - the logger service
  • nextRunnerGet - returns the next task runner defined in the job
  • processTask - this is the stub function called by the server that should be overloaded in each runner
  • raiseNextTask - starts the next task runner defined in the job
  • stateSave - saves the state on the current task
  • task - the config object for this task runner
  • taskService - the TaskService object
  • toCamelCase - converts a string with spaces to camel case
  • wait - simple sleep function

Task service

  • cache - an optional object that implements the CacheProvider interface
  • lookupAdd - adds a lookup. Set encrypted to true if it is an encrypted lookup
  • lookupDelete - deletes a lookup
  • lookupGet - gets a lookup
  • lookupGetValue - gets the value stored on a lookup
  • lookupUpdate - updates a lookup. Set encrypted to true if it is an encrypted lookup.
  • stateSave - saves the state on the current task
  • taskRaise - general method for raising a new task. Multiple examples in existing runners

Logger-service

Use this.logger.verbose('...'), this.logger.debug('...'), etc. to log information to the verbose, debug, log, warn and error log levels. These levels are used to stay lin line with those used in Nest.js.

Lookups

Lookups provide a way of storing values in an in-memory cache or the database for later use. Use the lookup functions on the task service.

  • lookupAdd - adds a lookup. Supplying a time to live value will use an in-memory cache if one is available.
  • lookupDelete - deletes the specified lookup
  • lookupGet - returns the specified lookup
  • lookupUpdate - updates the specified lookup
