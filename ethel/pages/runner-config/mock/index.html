Mock runners

These runners are generally used in testing and getting to know how ethel works. They don't contact any source or target information, they just update the message object and generate other tasks if other runners are configured.

The runners are:

  • MockExtractor
  • MockTransformer
  • MockLoader

MockExtractor

Demo only

This runner mocks extracting objects from a source system. It has the following configuration options:

  • generateError number - whether to generate a random error. The smaller the number the more likely an error. 1 gives an error every time.
  • messages number - how many tasks to create, defaults to 10
  • randomiseWait boolean - whether to randomise the configured wait value, defaults to true
  • wait number - milliseconds to wait before raising the next extract task

Example runner configuration:

  {
    "type": "MockExtractor",
    "config": {
      "generateError": 10,
      "messages": 100,
    }
  }

Tasks raised by this runner will have empty message objects.

MockTransformer

Demo only

This runner mocks transforming the message. It has the following configuration options:

  • generateError number - whether to generate a random error. The smaller the number the more likely an error. 1 gives an error every time.
  • randomiseWait boolean - whether to randomise the configured wait value, defaults to false
  • wait number - milliseconds to wait before raising the next extract task

Example runner configuration:

  {
    "type": "MockTransformer",
    "config": {
      "generateError": 10
    }
  }

Tasks raised by this runner will have empty message objects.

MockLoader

Demo only

This runner mocks loading objects into a target system. It has the following configuration options:

  • addHash boolean - whether to add the currentHash lookup, defaults to true
  • generateError number - whether to generate a random error. The smaller the number the more likely an error. 1 gives an error every time.
  • randomiseWait boolean - whether to randomise the configured wait value, defaults to false
  • wait number - milliseconds to wait before finishing the load step, defaults to 0

Example runner configuration:

  {
    "type": "MockLoader",
    "config": {
      "generateError": 10,
      "randomiseWait": true,
      "wait": 1000
    }
  }

It will raise the next task with the existing message object unchanged.
